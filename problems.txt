ANALISI BUG - Inference/test_*.py

────────────────────────────────────────────────────────────────────────────────
CRITICI (causano crash o risultati sbagliati)
────────────────────────────────────────────────────────────────────────────────

1. test_matrix_xor.py riga 643 — risultato di run_matrix_experiment mai aggiunto a all_results
   run_matrix_experiment(2) ritorna un dict ma non viene salvato in all_results.
   save_global_summary(all_results) viene chiamata con lista vuota → no-op silenzioso.
   Fix: all_results.append(run_matrix_experiment(2))

2. dataset_manager.py riga 122 — condizione if sempre True
   if mode == "repeat" or "random" or "zeros":
   "random" e "zeros" sono stringhe truthy → la condizione è sempre True.
   Qualsiasi stringa non-vuota supera il check, la branch elif mode == "function"
   è irraggiungibile e ValueError non viene mai sollevato.
   Fix: if mode in ("repeat", "random", "zeros"):

3. test_comparison_SZPvsTorch.py righe 177+182 — confronto sleale SZP vs PyTorch
   - PyTorch usa SGD invece di Adam (usato in tutti gli altri test)
   - PyTorch usa MSELoss invece di BCEWithLogitsLoss (usato ovunque)
   - Nessuna standardizzazione applicata in train_szp né train_pytorch,
     a differenza di tutti gli altri test
   Il confronto non è fair e i risultati non sono comparabili.

4. test_datasetsUCI.py riga 185 — log ogni singolo epoch
   if epoch % 1 == 0:   # è SEMPRE True: n % 1 == 0 per qualsiasi n
       logger.info(...)
   Probabilmente inteso come epoch % 10 == 0. Produce output massiccio.

5. test_datasetsUCI.py riga 94 — SyntaxError in Python < 3.12
   f"Model: size={"auto" if params['model_size'] == -1 else params['model_size']}, ..."
   F-string con virgolette annidate non è valida prima di Python 3.12.
   Fix: estrarre la sottoespressione in una variabile prima della f-string.

6. test_datasetsUCI.py righe 422+ — shadowing della variabile k
   k = params['k_folds']   # k = 5 (int, numero di fold)
   ...
   for k in METRICS:       # k viene sovrascritto → dopo il loop k == 'auc' (str)
   Se k viene usato come numero di fold dopo questo loop, il valore è sbagliato.

SIGNIFICATIVI (logica scorretta o inconsistente)

7. test_datasetsUCI.py righe 204 e 346 — final_accuracy non calcolata da final_probs
   final_accuracy = validation_accuracies[-1]   # ultimo epoch del training loop
   # poi final_probs viene calcolata ma NON usata per l'accuracy finale
   Logicamente inconsistente: final_accuracy e final_probs possono divergere
   in scenari edge. Anche training_time viene calcolato ma mai restituito né usato.

8. test_xor.py riga 701 — log con parametro sbagliato
   logger.info(f"Num Ising Perceptrons: {params['num_ising_1']}")
   Ma train_network_1L usa params['num_ising_net']. Il log mostra il valore errato.

9. test_xor.py riga 364 — commento fuorviante
   num_ising_perceptrons=params['num_ising_net'],  # Use num_ising_1 as the number of perceptrons
   Il commento dice num_ising_1 ma il codice usa num_ising_net.

10. test_xor.py righe 504-510 vs 807-815 — FullIsingModule creato con parametri diversi
    train_full_ising_model NON passa num_reads e num_workers a FullIsingModule.
    test_xor_2d_full_ising_module li passa entrambi.
    Stesso modello, comportamento diverso nelle due funzioni.

────────────────────────────────────────────────────────────────────────────────
MINORI (robustezza e code smell)
────────────────────────────────────────────────────────────────────────────────

11. Tutti i file tranne test_matrix_xor.py — os.getenv() senza valori default
    'random_seed': int(os.getenv('RANDOM_SEED'))  # crash TypeError se mancante
    test_matrix_xor.py fornisce default corretti per tutti i parametri.
    Gli altri file crashano con TypeError: int() argument must be... 'NoneType'.

12. test_xor.py e test_datasetsUCI.py — pattern global logger
    Il logger module-level viene riassegnato dentro funzioni con "global logger".
    Se le funzioni vengono chiamate in ordine diverso, il logger punta alla
    directory sbagliata.

13. test_comparison_SZPvsTorch.py riga 148 — soglia arbitraria per SZP
    pred_binary = (preds >= 0.5).astype(int)
    L'energia del modello Ising è tipicamente negativa. La soglia >= 0.5
    non è appropriata per energie Ising e non è giustificata nel codice.

14. test_matrix_xor.py — _GLOBAL_LOGGER calcolato al momento dell'import
    La directory di output con timestamp viene creata all'import del modulo,
    non all'esecuzione. Se il modulo viene importato e poi eseguito ore dopo,
    i log finiscono in una directory con timestamp sbagliato.

15. test_datasetsUCI.py e test_xor.py — dotenv.load_dotenv() senza path esplicito
    Carica il .env dalla CWD. Se lo script viene eseguito da una directory
    diversa da Inference/, il file .env non viene trovato e tutti i getenv()
    restituiscono None (crash silenzioso o TypeError).
    test_matrix_xor.py usa correttamente: dotenv_path=Path(__file__).parent / '.env'

16. test_comparison_SZPvsTorch.py riga 24 — import di SZP_Model potenzialmente mancante
    SZP_Model è un modulo legacy non presente nella struttura corrente del repo.
    L'import lancia ModuleNotFoundError all'avvio del file.

17. test_xor.py riga 988 / test_datasetsUCI.py — Wilcoxon con n troppo piccolo
    Con solo 6 dimensioni XOR (o pochi dataset), il Wilcoxon signed-rank test
    non ha potenza statistica sufficiente. Con n <= 5 il p-value non è affidabile.

18. test_xor.py — _save_metrics_csv con nanstd su singolo valore
    Nel confronto senza K-Fold, ogni metrica ha un solo valore per dimensione.
    np.nanstd([x]) restituisce sempre 0.0, rendendo la colonna _std nel CSV
    inutile e potenzialmente fuorviante.
